\subsection{Abstract}\label{abstract}

Conversational AI output tends to include content beyond what the
questioner is actually concerned with. We report on \textbf{LayerForge},
a deterministic pipeline that decomposes a corpus into 4±1 thematic
layers and returns only the layer(s) relevant to a query. The pipeline
combines five established components --- Cowan\textquotesingle s
working-memory upper bound, Newman modularity for community detection,
SCA-style per-layer distillation, HERCULES-style hierarchical KMeans,
and a deterministic core with no LLM in the analysis loop --- chosen so
that "AI noise filtering" does not itself depend on an AI. We
re-implemented a CPM (Constant Potts Model) backend (MIT-pure,
self-implemented) to test whether the theoretically
resolution-limit-free alternative would outperform Newman in this
small-N text domain (operational target N=20-40, external scaling check
up to N=100 on 20 Newsgroups). Across 119-row dual-method correlation
sweep, 196-cell N×K heatmap, 95-row ARI/NMI comparison, and the external
20-Newsgroups benchmark, \textbf{the Newman backend exceeded CPM by a
large effect size in our test range} (H\_struct exact match 87.5\% vs
19\%; 20NG ARI 0.430 vs 0.239 at default K=3, Newman reaches 0.557 when
K=4 is forced; same direction at N=100), against the naïve theoretical
expectation that CPM should win. \textbf{Positioning}: LayerForge shares
its core substrate (sentence-embedding similarity + community detection)
with prior work in clustering-based topic modeling, notably G2T (Zhang
et al. 2023). The contribution of this paper is twofold: (i) the
implementation report of combining SCA decomposition, HERCULES
hierarchical skeleton, the 4±1 cognitive constraint, and a Mode A/B/C
operational layer, and (ii) the systematic empirical comparison of
Newman vs CPM in the small-N text regime, including a three-failure-mode
mechanism analysis (§5). Concurrently with this work, Bohdal et al.
(Samsung 2026) independently arrived at the same problem space on the
LLM memory token side; we treat this convergence as evidence that the
design space is being identified by multiple independent groups. We
report an empirically derived method-selection rule (concept-only;
runtime implementation deferred because in the studied domain the rule
would output "newman" in essentially all cells). The artifact is
published as a Claude Code skill (LayerForge itself does not hold an API
key; LLM access at Boundary 1/2 is mediated through Claude Code), and is
reproducible end-to-end.

\subsection{§1. Motivation ---
関心毎に収束したい}\label{1-motivation--ux95a2ux5fc3ux6bceux306bux53ceux675fux3057ux305fux3044}

自然言語による AI との対話には、一見「便利」だが構造的に
\textbf{不利益}を生む特性がある:

\begin{quote}
\textbf{AI
は問いに答えるとき、関連する情報を包括的に提示しようとする}。だがユーザーが知りたい「関心」はその対話の瞬間において
\textbf{通常 1 つ}である(例: 「この PR の CI/CD
だけ知りたい」、「実験設計の前提だけ確認したい」)。
\end{quote}

包括的回答は次の三つの形でコストを生む:

\begin{enumerate}
\def\labelenumi{\arabic{enumi}.}
\item
  \textbf{読み手の認知負荷}
  人が一度に保持できる意味単位は短期記憶の容量に縛られる。Cowan (2010)
  は中心的な working memory capacity を 3-5 chunks と推定している。AI
  出力がこれを超えて「関心外の話題」も含むと、読み手は
  \textbf{関心に該当する部分の抽出に追加の認知コスト} を払う。
\item
  \textbf{multi-agent 連携の帯域浪費} AI agent A の出力を AI agent B
  が下流で処理する場合、A の出力が包括的だと \textbf{B は A
  の出力全体から関心領域だけ拾う}コストを払うか、または
  \textbf{関心外の情報に引きずられて drift する}かのどちらかになる。
\item
  \textbf{AI コストの線形浪費} full context
  を毎ターン処理することで、token
  単価ベースのコストが線形に上昇する。関心毎の本質情報量は full context
  より小さいことが多いため、本来不要な計算が含まれる。
\end{enumerate}

これら 3 つは独立な現象だが、**AI
出力が「関心毎に収束していない」**ことが \textbf{重なる原因} として 3
者すべてに寄与する (token コストは context window
全体の再処理にも依存するため source
は完全一致ではないが、関心毎収束が部分的に解になる)。LayerForge
は出力の収束を介して 3 者すべてに同時に影響を与えうる。

\subsubsection{1.1 Research question}\label{11-research-question}

\begin{quote}
AI の出力を \textbf{関心毎に対応する layer に収束させる}
ことで、上記三つのコストを同時に削減できるか? また、その「layer 化」を
\textbf{AI を再投入せずに} 決定論的に行えるか?
\end{quote}

第二の問いが重要である。\textbf{AI 出力を AI
で絞ると、同じ入力に対し異なる出力が得られる}(LLM
の確率的挙動)ため、関心毎収束の再現性が失われる。本研究は
\textbf{入力境界での AI
介在のみを許し、絞り込み層自体は決定論的に実装}する。

なお本稿で「決定論」と呼ぶのは \textbf{解析コア (R3 で詳述) の性質}
であり、入力境界 (Boundary 1: 自然言語 → 構造化)・出力境界 (Boundary 2:
構造化 → 自然言語) で optional に介在する LLM は本性質に含まれない。LLM
介在を含めた end-to-end の決定論性は、Boundary を mechanical fallback
に切り替えた場合に成立する (§3.1 参照)。

\subsubsection{1.2 Related work and
positioning}\label{12-related-work-and-positioning}

LayerForge の design space (sentence embedding + community detection on
similarity graph + per-cluster representation + clustering-driven
context compression) は周辺研究と重なる部分が多く、prior art に対する
position 整理が必要である。以下、隣接する 4 群の主要文献を整理し、最後に
LayerForge の差別化点を明示する。

\paragraph{1.2.1 Clustering-based topic modeling and graph-based text
clustering}\label{121-clustering-based-topic-modeling-and-graph-based-text-clustering}

文書/passage embedding を clustering し topic / cluster
構造を取り出す系列:

\begin{itemize}
\tightlist
\item
  \textbf{BERTopic} (Grootendorst 2022, arXiv:2203.05794) ---
  transformer-based document embedding + UMAP 次元圧縮 + HDBSCAN
  clustering + class-based TF-IDF。Per-document \textbf{single topic}
  assignment が default。
\item
  \textbf{Top2Vec} (Angelov 2020, arXiv:2008.09470) --- Doc2Vec /
  Sentence Transformer joint embedding + UMAP + HDBSCAN。topic
  数は自動決定。
\item
  \textbf{CETopic} (Zhang, Fang, Chen, Namazi-Rad, NAACL 2022, "Is
  Neural Topic Modelling Better than Clustering?") --- SimCSE embedding
  + UMAP + K-means + TF-IDF×IDFi variant。Neural topic model
  系列との直接比較を実施。
\item
  \textbf{Vec2GC} (Rao \& Chakraborty 2021, arXiv:2104.09439) ---
  term/document の weighted similarity graph 上の community
  detection、density-based + hierarchical clustering をサポート。
\item
  \textbf{G2T (Graph2Topic)} (Zhang, Liu, Yan 2023, arXiv:2304.06653)
  --- pretrained LM embedding → semantic graph 構築 → community
  detection で topic identification → TFIDF variant で word-topic
  distribution。\textbf{本研究の design substrate に最も近い prior
  art}。Automatic evaluation で BERTopic / CETopic に対し SOTA を主張。
\item
  \textbf{SCA} (Eichin et al. 2024, arXiv:2410.21054) --- clustering
  の上で \textbf{per-document multi-topic distribution} を導入、単純
  clustering の単一所属仮定を緩和。本研究の R3 (distillation) の元実装。
\item
  \textbf{LLM-Assisted Topic Reduction for BERTopic} (Janssens, Bogaert,
  Van den Poel 2025, arXiv:2509.19365) --- BERTopic の出力に LLM を
  merge step として追加する hybrid 系列。本研究の決定論性 (R5)
  と方向が異なる。
\end{itemize}

これらは「\textbf{document/passage を topic 群に分類する}」task
formulation を取る。BERTopic / Top2Vec / CETopic は density-based
(HDBSCAN) または K-means が主流で community detection は使わず、Vec2GC /
G2T が graph-based community detection 系列。\textbf{G2T
は本研究と最も近い substrate} だが、community detection algorithm 名
(Louvain / CPM / Leiden) が abstract レベルで明示されておらず、Newman vs
CPM の systematic 比較は内部で行われていない (本研究の §4-§5
の独自性が残る差)。

\paragraph{1.2.2 Hierarchical RAG and
GraphRAG}\label{122-hierarchical-rag-and-graphrag}

階層構造 + retrieval / summarization の組合せ系列:

\begin{itemize}
\tightlist
\item
  \textbf{RAPTOR} (Sarthi et al. 2024, ICLR) --- community detection +
  LLM summarization で hierarchical RAG。
\item
  \textbf{GraphRAG} (Edge et al. 2024, arXiv:2404.16130) --- Leiden +
  LLM summarization で knowledge graph 構造化、query-focused
  summarization。
\item
  \textbf{HippoRAG} --- PageRank-based retrieval、graph 構造を index
  側に使う。
\item
  \textbf{Core-based Hierarchies for Efficient GraphRAG} (Hossain \&
  Sarıyüce 2026, arXiv:2603.05207) --- GraphRAG の Leiden clustering を
  \textbf{k-core decomposition} に置換し deterministic な hierarchy
  を得る。Leiden 批判の根拠が "modularity optimization admits
  exponentially many near-optimal partitions on sparse graphs"
  で、本研究が §5 で引用する Good (2010) Q degeneracy と
  \textbf{同じ問題を別 angle (置換) で攻める} 関係。LayerForge は
  modularity family 内に留まり failure mode を documents する方向。
\item
  \textbf{RAG vs. GraphRAG: A Systematic Evaluation} (Han et al. 2026,
  arXiv:2502.11371) --- RAG / RAPTOR / HippoRAG2 / GraphRAG
  の統一評価フレームワーク提案。LayerForge は本 framework の「passage
  graph + decoupled summarization」象限に位置する。
\item
  \textbf{GraphRAG-Bench} (Xiao et al. 2025, arXiv:2506.02404) ---
  GraphRAG family の domain-specific reasoning benchmark。
\item
  \textbf{HERCULES} (\texttt{bandeerun/pyhercules}、MIT 2025) ---
  LLM-integrated hierarchical KMeans の reference
  implementation。peer-reviewed publication
  は本研究時点で未刊行、本稿は引用 base ではなく \textbf{設計 skeleton
  として参照} する位置付け (§2.4)。
\end{itemize}

これらは大半が \textbf{解析 pipeline に LLM を組み込む}
前提を持つ。LayerForge は決定論性 (R5) を理由に \textbf{解析コアから LLM
を排除}、LLM 介在を Boundary 1/2 (optional) に限定する点が architectural
な差。Core-based GraphRAG とは "Leiden の reproducibility 問題への対応"
という problem 認識を共有しつつ、置換 (k-core) ではなく \textbf{failure
mode の documentation + method-selection signal}
という解決方向で分岐する。

\paragraph{1.2.3 Context and memory
compression}\label{123-context-and-memory-compression}

LLM の context budget 制約下で input を圧縮する系列 (近年の直近系研究):

\begin{itemize}
\tightlist
\item
  \textbf{Clustering-driven Memory Compression for On-device LLMs}
  (Bohdal, Saha, Michieli, Ozay, Ceritli 2026, arXiv:2601.17443) ---
  \textbf{LayerForge と framing が最も近い同時期 prior art}。"groups
  memories by similarity and merges them within clusters prior to
  concatenation" という mechanism は本研究の Mode C (multi-agent
  帯域削減) と同型。ただし対象は \textbf{user-specific memory tokens}
  (personalization 用) であり、目的は context budget 制約下の
  personalization 維持。LayerForge とは「対象 (memory vs corpus
  passage)、目的 (personalization vs 関心毎収束)、cognitive constraint
  (なし vs Cowan 4±1 明示)、method (similarity averaging-then-merge vs
  community detection on similarity graph)」の 4 点で differentiate
  される (§1.2.5 で詳述)。
\item
  \textbf{EDU-based Context Compressor} (Zhou et al. 2025,
  arXiv:2512.14244) --- Elementary Discourse Unit decomposition による
  structure-then-select 圧縮。compression mechanism が clustering
  ではなく discourse 構造解析という点で本研究と直交。
\end{itemize}

本系列は本研究と「\textbf{clustering-driven context compression}」という
framing を共有する。本研究の \textbf{§1 の動機 (認知負荷 / 帯域 / コスト
3 軸)} と Bohdal et al. の "balances context efficiency and
personalization quality" が独立に同じ problem space
に到達している事実は、本研究が単独の design ではなく \textbf{2025-2026
年に複数研究者が並行して取り組む problem space に位置する}
ことを意味する。

\paragraph{1.2.4 Community detection method
comparisons}\label{124-community-detection-method-comparisons}

複数の community detection 手法を benchmark で systematic 比較 /
理論分析する系列:

\begin{itemize}
\tightlist
\item
  \textbf{Lancichinetti \& Fortunato (2009)} --- LFR synthetic benchmark
  を提案、community detection algorithm の標準的評価基盤。
\item
  \textbf{Aldecoa \& Marín (2013, Scientific Reports)} --- Newman
  modularity / CPM / RB / RN / MCL / Infomap 等を closed benchmark
  で系統評価、CPM が「benchmark
  の両端の易しい領域でのみ精度が出る」「ある程度の不安定性を示す」と報告。本研究の
  §4-§5 の small-N text 結果はこの synthetic
  結果との対比軸として位置付け可能。
\item
  \textbf{Traag et al. (2011, 2019)} --- CPM の resolution-limit-free 性
  (2011)、Leiden refinement (2019) の理論基礎。
\item
  \textbf{From Leiden to Pleasure Island: The Constant Potts Model as a
  Hedonic Game} (Felipe, Avrachenkov, Menasche 2025, arXiv:2509.03834)
  --- CPM を hedonic game (potential game) として再定式化、local utility
  最適化が pseudo-polynomial time で equilibrium
  に収束することを証明、robust partition criteria
  を導入。\textbf{本研究の §5 (CPM の failure mode 分析) と直近の関連}
  だが、彼らは synthetic / random graph ベース、本研究は small-N text
  similarity graph で empirical に観察を取る相補的関係。
\end{itemize}

これらは合成 graph / closed benchmark /
理論分析が中心であり、\textbf{small-N text similarity graph (LayerForge
の運用域 N=20-40) における Newman vs CPM systematic 比較}
は本研究時点では我々が見つけられた範囲では他に存在しない。

\paragraph{1.2.5 LayerForge
positioning}\label{125-layerforge-positioning}

上記 prior art に対し本研究の差別化点は以下 5 点:

\begin{enumerate}
\def\labelenumi{\arabic{enumi}.}
\tightlist
\item
  \textbf{vs G2T (Zhang et al. 2023)}: 共通 substrate (sentence
  embedding + community detection on similarity graph)
  を持つが、本研究は \textbf{Newman vs CPM の small-N text 領域での
  systematic 比較 + 3 failure mode 分類 (§5)} に focus、加えて
  \textbf{SCA distillation step (R3) と HERCULES skeleton (R4)、4±1
  cognitive constraint (R2)、Mode A/B/C application layer} を組合せ。G2T
  との直接 head-to-head 比較は本稿の scope 外 (future work)。
\item
  \textbf{vs Clustering-driven Memory Compression (Bohdal et al. 2026,
  Samsung)}: clustering-driven context compression という最上位 framing
  は完全に一致する同時期 prior art。差別化は以下 4 軸 --- (i)
  \textbf{対象}: LayerForge は corpus passage、Bohdal は LLM 内部の
  user-specific memory token、(ii) \textbf{目的}: LayerForge は AI
  output の関心毎収束 (認知負荷 + 帯域 + コスト 3 軸削減)、Bohdal は
  personalization 維持下の context budget 適合、(iii) \textbf{method}:
  LayerForge は community detection on similarity graph (Newman / CPM
  切替)、Bohdal は similarity averaging + merge、(iv) \textbf{cognitive
  constraint}: LayerForge は Cowan (2010) 4±1 を ergonomic
  上限として明示採用、Bohdal は context budget
  による外部制約のみ。両研究は \textbf{同じ problem space
  に独立に到達した近接研究} であり、本稿は corpus-passage 側で
  community-detection 系の手法選択 (Newman vs CPM) を実証評価する
  complementary な貢献を提供する。
\item
  \textbf{vs BERTopic / Top2Vec / CETopic}: 本研究は \textbf{per-passage
  hard partition} を一次成果物とし、layer 構造の上で
  \textbf{deterministic Mode A/B/C} を提供する。BERTopic 系列は
  per-document topic distribution を一次成果物とする task formulation
  の違いから、direct baseline 比較対象には適さない (§2.1)。
\item
  \textbf{vs RAPTOR / GraphRAG / Core-based GraphRAG / HERCULES}:
  本研究は \textbf{解析コアから LLM を意図的に decouple} し、Boundary
  1/2 に限定する (R5)。これは関心毎収束の \textbf{再現性保証}
  を目的とした design choice であり、LLM
  統合系列とは正反対の方向。Core-based GraphRAG と "Leiden の
  reproducibility 問題" の認識を共有しつつ、置換 (k-core) ではなく
  \textbf{modularity family 内での failure mode documentation +
  method-selection signal 構成} という方向で分岐。
\item
  \textbf{vs Aldecoa \& Marín / Traag / Felipe et al. (Pleasure
  Island)}: 既存の community detection 比較は synthetic / random /
  closed benchmark 中心で、small-N text domain
  での「resolution-limit-free が機能するか」の empirical
  検証は本稿で初めて (我々が確認できた範囲で)。本研究の §4 / §5
  の貢献の核がここにある。
\end{enumerate}

つまり LayerForge は \textbf{G2T の design space に SCA + HERCULES + 4±1
+ Mode A/B/C を載せた application} であり、core 貢献は \textbf{(i)
その組合せの実装報告、(ii) Newman vs CPM の small-N text 領域での
systematic comparison + 3 failure mode 分類 (§5)} の 2
点に絞られる。「community detection method comparison そのものを新規
contribution と主張する」のではなく、「\textbf{既知の method 群を本
domain で実証評価して挙動を文書化した}」と読まれるべき report
である。Bohdal et al. (Samsung 2026) との同時期 framing 重複は本研究の
novelty を否定するものではなく、\textbf{両研究が独立に同じ problem space
を identify した事実} として記録する。

\subsubsection{2.1 (R1) なぜ community detection
か}\label{21-r1-ux306aux305c-community-detection-ux304b}

「関連する文の集合」を \textbf{教師なしで}
自動抽出する手法として確立済み。RAG 拡張 (RAPTOR, GraphRAG, HippoRAG)
も同類の発想で passage clustering に Louvain/Leiden
を使う。本研究はその一族だが、後述 (R5) の制約から \textbf{community
detection を LLM summary と decouple} している点が異なる。

なお、document-level の topic 分布を出す手法 (LDA, BERTopic 等の topic
modeling 系列) は \textbf{passage-level の hard partition
を直接出すのが本来の目的でない} ため、本研究の「各 passage がどの layer
に属するか」という task formulation と直接比較する baseline
には適さない。本稿の比較対象は \textbf{community detection method
内部の選択 (Newman vs CPM)} に限定する。

\subsubsection{2.2 (R2) なぜ 4±1 か}\label{22-r2-ux306aux305c-41-ux304b}

Cowan (2010) は短期記憶の中心的容量を 3-5 chunks
と推定。「関心毎収束した結果が読み手に易しい」と要求するなら、layer 数の
\textbf{ergonomic な上限} をここに置くのが自然。これは「4±1 が universal
な認知定数」と主張するのではなく、\textbf{設計の目標値として採用}するというデザイン判断。

\subsubsection{2.3 (R3) なぜ SCA distillation
か}\label{23-r3-ux306aux305c-sca-distillation-ux304b}

community detection は「どの文がどの layer に属するか」を出すが、layer
の \textbf{代表表現} までは出さない。読み手 / 下流 agent が layer
に渡すには「この layer は何の話題か」が短く分かる形が必要。Eichin et al.
(2024) の Semantic Component Analysis は per-cluster の代表 component を
distill する手法で、本要件と整合する。

\subsubsection{2.4 (R4) なぜ HERCULES
か}\label{24-r4-ux306aux305c-hercules-ux304b}

flat 1-level community detection
で本研究の主要結果は十分得られるが、将来的に「N=10K の large corpus を
4×4×4×4 = 256
のような階層に再帰展開する」運用を想定し、\textbf{hierarchical KMeans の
skeleton} を予め採用 (現実装は max\_depth=1 default、option で recursive
化可能)。

なお HERCULES は本研究時点では \textbf{GitHub の reference
implementation (\texttt{bandeerun/pyhercules}、MIT) のみ存在し
peer-reviewed publication なし}。本研究は HERCULES を「hierarchical
KMeans を採用した先行 OSS implementation」として position
参照しており、HERCULES 自体の理論的優位性を引用 base
にしているわけではない。R4 の本質的根拠は「再帰的 KMeans が large corpus
拡張に skeleton として機能する」という設計判断であり、HERCULES
はその一実装例 (=参照先) という位置付け。

\subsubsection{2.5 (R5)
なぜ決定論コアか}\label{25-r5-ux306aux305cux6c7aux5b9aux8ad6ux30b3ux30a2ux304b}

§1 で述べた通り、\textbf{LLM
の確率的挙動は関心毎収束の再現性を破壊する}。同じ入力で異なる layer
化が起きたら「収束後の出力」を信用できない。本研究は:

\begin{itemize}
\tightlist
\item
  \textbf{入力境界}: 自然言語 → 構造化 (Boundary 1) で LLM 介在可
\item
  \textbf{解析コア}: 決定論 (community detection + KMeans + SCA
  は全て決定論アルゴリズム)
\item
  \textbf{出力境界}: 構造化 → 自然言語 (Boundary 2) で LLM 介在可
  (optional、本研究では Mode A/C は AI 不要で動作)
\end{itemize}

この制約により、解析結果は \textbf{同じ input + 同じ hyperparameters →
同じ output} を保証する。ここで hyperparameters には community-detection
method の選択
(\texttt{community\_method="newman"\ \textbar{}\ "cpm"})、random
seed、target\_range、embedding model
等を含む。これらを固定した上での再現性であり、method
を切り替えれば異なる partition が得られうる (§4.3 で実証)。

\subsubsection{2.6 まとめ ---
組合せの必然性}\label{26-ux307eux3068ux3081--ux7d44ux5408ux305bux306eux5fc5ux7136ux6027}

5 つの技術選択はそれぞれ独立の要件 (R1-R5)
から導かれており、\textbf{逆に言えば各要件を満たす別の選択は本研究の課題に合わない}:

\begin{itemize}
\tightlist
\item
  community detection を skip → 関心単位の自動抽出ができない
\item
  4±1 を捨てる → 認知負荷削減の根拠を失う
\item
  SCA を skip → 各 layer の代表表現が出ない
\item
  HERCULES を捨てる → 大規模拡張の base がない
\item
  決定論を捨てる → §1 の問題を AI で AI を絞ろうとして自己矛盾
\end{itemize}

この組合せが §1 の動機 (R1-R5) を実装上充足する \textbf{設計根拠}を §3
で示す。組合せ内部の \textbf{最重要選択 (community detection method の
Newman vs CPM)} の妥当性を §4 で empirical に検証する。\textbf{SCA
distillation / HERCULES hierarchy / 4±1 制約 / 決定論性の個別 ablation
は本稿の scope 外} であり、§7.5
で明示する。本稿の主張は「組合せ内部の最重要選択について empirical に
method 比較を行い、本 domain で Newman backend
が適合することを示した」に限定される。

\subsubsection{4.4 H\_struct (Q peak K = n\_themes)

本 sub-section は §4.1 とは \textbf{異なる軸} を測る:

\begin{itemize}
\tightlist
\item
  §4.1: 「\textbf{同一 corpus 構造で N が変動した時} Q peak K
  がどう動くか」(Q degeneracy の reproduction)
\item
  §4.4: 「\textbf{異なる n\_themes corpus でそれぞれ} Q peak K が
  n\_themes に一致するか」(H\_struct hypothesis の test)
\end{itemize}

16 settings = \textbf{4 n\_themes (3, 4, 5, 7) × 2 embedders (MiniLM,
mpnet) × 2 seeds (42, 123)}。各 setting で K range を
\texttt{\{1-2,\ 2-3,\ 3-4,\ 3-5,\ 5-7,\ 6-8,\ 8-10,\ 10-12\}} に
sweep。range の構成意図は二段: (a) 全 K=1 から 12 を
\textbf{重複を許して連続 cover} することで \texttt{find\_valid\_scale}
の binary search 安定性を K 域全体で確認、(b) Cowan 4±1 (=3-5) 周辺は
\texttt{2-3,\ 3-4,\ 3-5} の \textbf{3 ranges で密に cover}
することで本研究の関心域 (4±1) での誤差を小さくする。各 range で Q
が最大となる K\_actual を取得し、対応する \texttt{n\_themes} と比較:

\begin{itemize}
\tightlist
\item
  Newman: \textbf{14/16 (87.5\%) 完全一致}
  (\texttt{K\_actual\ ==\ n\_themes})
\item
  CPM: \textbf{3/16 (19\%) 完全一致}
\end{itemize}

\textbf{観察}: 「Q peak K が corpus の自然な theme 数を tracking
する」という仮説 (H\_struct) は \textbf{本 16 settings (synthetic
disjoint-vocabulary corpus) の domain で Newman backend に強く支持}
され、CPM backend では支持されなかった。なお 16 settings は small sample
のため、binomial test 等の formal
な統計的有意性検定は本稿では実施せず、効果サイズ (87.5\% vs 19\%)
の大きさで定性的に評価する。

\textbf{Scope 限定 (重要)}: 上記 87.5\% tracking は \textbf{synthetic
disjoint-vocabulary corpus 上での結果} であり、「Newman の
\texttt{find\_valid\_scale} pipeline が任意 corpus で n\_themes を自動
tracking する」という強い主張は本稿では行わない。External benchmark
(§4.6 で示す 20 Newsgroups、N=100) では default の
\texttt{target\_range=(3,5)} 設定で Newman ですら K=3 を返し n\_themes=4
を tracking 失敗する。これは disjoint-vocabulary という synthetic
条件と公開 corpus の vocabulary overlap の差を反映したもので、§5.4
(find\_cpm\_resolution 同型の calibration 設計が target\_range 下端 K
を選好する点) で機序的に位置付ける。本節の H\_struct 主張は
\textbf{synthetic corpus 条件下での部分支持} と読まれるべきである。

→ \textbf{Figure}: \texttt{figures/new/fig\_h\_struct.png} --- 2-panel
scatter (Newman \textbar{} CPM) で K\_actual vs n\_themes、対角線が
perfect tracking を示す。Newman side は対角線周辺に集中、CPM side は
systematically 下方に外れる (under-merging を視覚化)。

family を用いる:

clusters than themes」へ強制される}ため、ground truth とは別に partition
の細分化が支配し method 差は出にくくなる。粒度域の違いが method
差の顕在化を決める。

\subsection{§5. Why CPM Underperforms --- 三つの failure mode
と運用方針}\label{5-why-cpm-underperforms--ux4e09ux3064ux306e-failure-mode-ux3068ux904bux7528ux65b9ux91dd}

§4 で観察された結果のうち最も予想と異なったのは \textbf{CPM が Newman
より一貫して劣る} ことである。Traag et al. (2011) は CPM が
resolution-limit-free であることを subgraph-invariance
性質で数学的に示しており、Newman の Q degeneracy (Good 2010) に対する
\textbf{正しい修正} とされる。にもかかわらず本研究の small-N text domain
では CPM が逆方向に劣化した。本節は \textbf{三つの異なる failure mode}
に切り分けて機序を整理する。Karate Club 上の K=4 over-split (§7.3) と 20
Newsgroups 上の K=3 under-merge (§4.6) という反対方向の挙動も、別 mode
として一貫に位置付けられる。

\subsubsection{5.1 三つの failure
mode}\label{51-ux4e09ux3064ux306e-failure-mode}

CPM-Louvain が ground truth から外れる原因は本研究の検証範囲で
\textbf{少なくとも三つの独立な機序} に分離できる:

\subsubsection{5.2
目的関数の構造差}\label{52-ux76eeux7684ux95a2ux6570ux306eux69cbux9020ux5dee}

両 method の objective function を比較する:

\begin{itemize}
\tightlist
\item
  Newman:
  \texttt{Q\ =\ (1/2L)\ Σ\_ij\ {[}A\_ij\ -\ (k\_i\ k\_j\ /\ 2L){]}\ δ(c\_i,\ c\_j)}

  \begin{itemize}
  \tightlist
  \item
    内部 edges を null model (期待値 k\_i k\_j / 2L) と比較
  \item
    cluster サイズに対する明示的 penalty なし
  \end{itemize}
\item
  CPM: \texttt{H\ =\ Σ\_c\ {[}m\_c\ -\ γ\ ×\ (n\_c\ choose\ 2){]}}

  \begin{itemize}
  \tightlist
  \item
    intra-edges から \textbf{クラスタペア数 × γ} を引く
  \item
    cluster size n\_c に対し \textbf{二次的に増大} する penalty
  \end{itemize}
\end{itemize}

→ \textbf{Figure}: \texttt{figures/new/fig\_cpm\_mechanism.png} ---
cluster size n\_c に対する penalty/null contribution
の推移を比較プロット。CPM の \texttt{(n\_c\ choose\ 2)} quadratic curve
が Newman null-model の線形 reference を超える "under-merging zone"
を網掛けで明示。

§4.2 で示した \textbf{common-reference} (両 method を corpus median
similarity の同一 graph 上で比較) では、N=12-40 の運用域で above-limit
fraction の差が 6 cells 中 5 cells で完全一致する。すなわち本 domain
では \textbf{Newman の resolution limit は operationally bind

が解こうとしている問題が \textbf{そもそも本 domain
では問題になっていない}。残るのは CPM 固有の penalty
構造の作用のみ、というのが §5.3 以降の出発点となる観察である。

\subsubsection{5.3 Failure mode (c) --- (n\_c choose 2) penalty effect
(small-N text
domain)}\label{53-failure-mode-c--n_c-choose-2-penalty-effect-small-n-text-domain}

LayerForge の運用域 (N=20-40 cross-domain text corpus) では:

\begin{itemize}
\tightlist
\item
  edge density は中程度 (sparse でも dense でもない)
\item
  §4.2 の common-reference data が示す通り、above-limit fraction が
  0.25-0.80 の範囲で推移し N ≥ 20 ではほとんど 0.5 以上 → Newman の
  resolution limit は operationally bind していない
\item
  CPM を適用すると \texttt{(n\_c\ choose\ 2)} 項が \texttt{m\_c}
  を相対的に圧倒、「大きい cluster は H 損」が dominant force
  となり、CPM は \textbf{systematically coarse な partition} を選ぶ
  (K-matched 比較でも残る under-merging)
\end{itemize}

これが \textbf{§4.4 (H\_struct 87.5\% vs 19\%)} と \textbf{§4.6 K=4
強制時の Newman 0.557 vs CPM 0.239} の数値ギャップの根本理由である。Mode
(c) は K を強制した状態でも残る成分であり、本 domain における Newman
優位の \textbf{構造的} 主因。

\subsubsection{5.4 Failure mode (b) --- find\_cpm\_resolution の
calibration
bias}\label{54-failure-mode-b--find_cpm_resolution-ux306e-calibration-bias}

本実装の \texttt{find\_cpm\_resolution} (および対応する Newman 側の
\texttt{find\_valid\_scale}) は target\_range 内で \textbf{smallest K を
preferred} する設計である。具体的に
\texttt{cpm\_backend.py::find\_cpm\_resolution} (L262-263) は:

\begin{Shaded}
\begin{Highlighting}[]
\ControlFlowTok{if}\NormalTok{ K\_min }\OperatorTok{\textless{}=}\NormalTok{ k }\OperatorTok{\textless{}=}\NormalTok{ K\_max:}
\NormalTok{    best }\OperatorTok{=}\NormalTok{ (mid, labels, h, k)}
\NormalTok{    hi }\OperatorTok{=}\NormalTok{ mid  }\CommentTok{\# prefer smaller γ (smaller K)}
\end{Highlighting}
\end{Shaded}

これは「より coarse な partition は (4±1 制約下で)
解釈しやすい」という運用上の判断 (working memory 上限が ergonomic
上限なら K は小さい方が望ましい) を反映した設計だが、副作用として
\textbf{n\_themes が target\_range{[}0{]} と一致しない corpus では
tracking 失敗} が生じる。

§4.6 (20 Newsgroups、n\_themes=4、target\_range=(3,5)) で両 method とも
K=3 に収束する現象はこの mode の symptom。K=4 を強制すると Newman は
ARI=0.557 まで上昇するが (mode (b) を回避できれば mode (c)
の効果のみ残る)、default 動作だけを見ると Newman もまた n\_themes を
tracking できない。

\textbf{§4.4 (synthetic corpus、K range が n\_themes 周辺を密に cover)
で Newman が 87.5\% tracking できる}のは、K range 設計が n\_themes を
target\_range{[}0{]} 近傍に置いており mode (b)
の影響が小さいため。External corpus に対しては default の
target\_range=(3,5) では mode (b) が顕在化することに本稿は注意を促す。

\subsubsection{5.5 Failure mode (a) --- Louvain refinement gap
(cross-ref to
§7.3)}\label{55-failure-mode-a--louvain-refinement-gap-cross-ref-to-73}

本実装の CPM-Louvain は vanilla Louvain (greedy single-node move)
であり、Leiden refinement (Traag 2019) は含まない。Karate Club のような
dense small graph では K=2 macro split が single-node move
の局所最適でなく、γ を低くしても K=4 sub-clustering に stuck する
(文献既知)。本稿の §7.3 で Karate Club の結果 (ARI 0.595, K=4) はこの
mode に該当する旨を明示する。

本実装の小規模 text domain での主要結果 (§4.4, §4.6) は dense small
graph でないため mode (a) は dominant でないが、\textbf{外部対比 (Karate
Club) で K が ground truth より大きく出る現象} はこの mode
で説明される。CPM の penalty
構造そのものとは独立の、algorithm-engineering 上の制約である。

\subsubsection{5.6 含意 --- "resolution-limit-free is universally
better" は domain
依存}\label{56-ux542bux610f--resolution-limit-free-is-universally-better-ux306f-domain-ux4f9dux5b58}

CPM の理論的優位は \textbf{resolution limit が binding な domain}
で発揮される。LayerForge の small-N text domain では §4.2
common-reference data が示す通り limit binding がそもそも稀で、CPM の
"修正" が \textbf{解こうとしている問題が本 domain
では問題になっていない}。同時に、CPM 固有の penalty 構造 (mode (c)) と
calibration 設計 (mode (b)) が \textbf{別の問題} を生み、結果として
Newman が一貫して上回る。

これは Traag et al. (2011) の理論結果 (CPM の subgraph-invariance)
への直接反論ではない。\textbf{operating domain の特性 (limit binding
の有無、典型的な n\_themes と target\_range の関係、graph の
dense/sparse 度合い) と照らした上で method 選択が必要} であるという
mechanism observation である。

\subsection{§6. Application examples}\label{6-application-examples}

LayerForge の Mode A/B/C は §1 の三つのコスト (認知負荷 / 帯域 / AI
コスト) にそれぞれ対応する応用域を持つ:

\subsubsection{6.1 Mode A ---
読み手の認知負荷削減}\label{61-mode-a--ux8aadux307fux624bux306eux8a8dux77e5ux8ca0ux8377ux524aux6e1b}

corpus を 4±1 layer に分解、UI / CLI で「興味のある layer
のみ」展開可能。reader は full corpus を読まずに、関心 layer
の代表表現から内容を判断できる。

\subsubsection{6.2 Mode B ---
意思決定情報の関心遷移管理}\label{62-mode-b--ux610fux601dux6c7aux5b9aux60c5ux5831ux306eux95a2ux5fc3ux9077ux79fbux7ba1ux7406}

考慮中の意思決定情報を layer
化し、\texttt{open}/\texttt{close}/\texttt{settle}
状態を持たせる。決定中の事項 (open) と決定済み事項 (settled)
を分離することで、読み手は \textbf{今活性化している関心} だけに focus
できる。

\subsubsection{6.3 Mode C --- multi-agent
帯域削減}\label{63-mode-c--multi-agent-ux5e2fux57dfux524aux6e1b}

AI agent A の verbose 出力 (例: 15K token の包括回答) を agent B の
query に関連する layer subset に圧縮 (decision-less subset
保証、つまり情報の subset としてのみ機能、要約は LLM
が行うわけでない)。本研究の \texttt{scripts/multiagent\_demo/} で 63\%
の context reduction を達成 (B 軸 null result: 圧縮後でも下流 agent
の出力品質は維持された)。

\textbf{§4.5 の 1/K=10\% との関係 (異なる measurement)}: §4.5
が示した「K=10 で 1/K=10\% (10x reduction)」は \textbf{partition
consistency 上の理論最小読込量比} (passage を query にした self-routing
が成功する前提で、所属 layer 1 つだけ読めば十分という上限)
であり、measurement の単位は「読み込む layer 数 / 全 layer 数」。本節
Mode C の 63\% reduction は \textbf{token 数ベースの実測値} で、demo
script で agent A の full output と agent B が実際に読み込む subset とを
token 単位で比較したもの。両者は \textbf{異なる measurement} (理論最小
vs production 実測) で直接比較できない。前者は K=10 の細粒度 partition
での floor、後者は Mode C の現実的な workflow での measured reduction。

\subsection{§7. Limitations}\label{7-limitations}

\subsubsection{7.1
検証範囲の限定}\label{71-ux691cux8a3cux7bc4ux56f2ux306eux9650ux5b9a}

\begin{itemize}
\tightlist
\item
  \textbf{運用想定は N=20-40 small-N text domain}。External benchmark
  として 20 Newsgroups で N=100 まで scaling check を実施
  (§4.6)、結論は同方向 (Newman 優位)。\textbf{§4.3 の ARI 表は
  N=12-40、§4.6 は N=100 で、中間 N (40-100) 域は本稿で直接 sweep
  していない} が、両端点で Newman
  優位が同方向に観察されたことから補間域での挙動も同方向と推測される
  (確認は future work)。\textbf{N \textgreater{} 1000 や dense graph
  は未検証}
\item
  \textbf{External benchmark は 20NG 1 件のみ} --- 他 public corpus
  での再現は future work
\item
  \textbf{CPM-Louvain は Leiden refinement なし} --- 文献既知の Karate
  Club K=4 sub-clustering 現象に該当、§7.3 で具体的に位置付け
\end{itemize}

\subsubsection{7.2 B 軸の null
results}\label{72-b-ux8ef8ux306e-null-results}

LayerForge を AI workflow に組込んだ際の \textbf{LLM 自体の behavior
改善} は本研究の範囲外で、実測した 3 試行 (hallucination
benchmark、multi-agent drift verification、context filter ablation)
はすべて \textbf{null result} だった。これは「LayerForge が AI
を賢くする」claim が defensible でないことを意味する。本研究の貢献は
\textbf{AI 出力を関心毎に絞り込んで読み手 / 下流 agent に渡す pipeline
自体} にあり、AI 行動の改善は claim しない。

\subsubsection{7.3 自前実装の correctness
gate}\label{73-ux81eaux524dux5b9fux88c5ux306e-correctness-gate}

CPM-Louvain の reference implementation である GPL \texttt{leidenalg}
との数値正確性 head-to-head 比較は MIT license
との互換性問題で実施せず。代替として以下の \textbf{3 段階の間接 gate}
を設けている:

\begin{enumerate}
\def\labelenumi{\arabic{enumi}.}
\tightlist
\item
  \textbf{Synthetic 3-block test}
  (\texttt{tests/axioms/test\_cpm\_backend.py::test\_cpm\_partition\_separates\_three\_blocks}):
  3 blocks (intra=0.9, inter=0.05) で K=3 を完全 recover
\item
  \textbf{Karate Club ARI = 0.595} vs empirical 2-community ground truth
  (chance baseline = 0): Zachary (1977) 公開 graph で、Leiden refinement
  不在による over-splitting (§7.1 で言及した K=4 sub-clustering 現象)
  のため partition の cluster 数を直接確認すると K=4 となる。これが
  ground truth の 2-community split の \textbf{sub-refinement}
  に該当することは cluster 構造から判定でき、ARI=0.595 は \textbf{この
  sub-refinement と ground truth の整合度} を示す中間値である
  (完全一致なら ARI=1.0、ランダムなら 0)。Leiden refinement を加えれば
  K=2 macro split に到達する余地があることを示唆
\item
  \textbf{Determinism cross-check}: 同 seed で Newman / CPM とも
  reproducible
\end{enumerate}

これは「CPM が正しく実装されている」の strong proof ではない (Leiden
refinement なしの local optimum 限界が §5.5 / §7.1
で言及済み)。「\textbf{Newman との empirical
比較を行う前提として最低限の correctness 要件を満たしている}」ことの
evidence として位置付ける。

\subsubsection{7.4 4±1 の universality
主張なし}\label{74-41-ux306e-universality-ux4e3bux5f35ux306aux3057}

Cowan (2010) の working memory capacity
は本研究の設計目標として採用したが、「4±1 が任意の corpus で universal
に emerge する」とは主張しない。実測では corpus に応じて K\_optimal は
2〜11 の範囲で変動 (§4.1)、ergonomic な上限としての位置付けに留める。

\subsubsection{7.5 Scope 外として宣言する
ablation}\label{75-scope-ux5916ux3068ux3057ux3066ux5ba3ux8a00ux3059ux308b-ablation}

本稿は \textbf{community detection method 選択 (Newman vs CPM)}
の妥当性検証に scope を限定した。組合せ内部の他の選択肢に関する以下の
ablation は実施していない:

\begin{itemize}
\tightlist
\item
  \textbf{SCA distillation on/off} の partition / output quality への
  effect
\item
  \textbf{HERCULES hierarchy depth = 1 (本稿の default) vs depth
  \textgreater{} 1} の effect
\item
  \textbf{4±1 制約 vs 制約なし (任意 K\_optimal)} の cognitive load
  削減効果差
\item
  \textbf{Boundary 1/2 で LLM 介在あり/なし} の output drift 測定
\end{itemize}

これらは LayerForge
の組合せ全体を本稿より広範に検証する場合に必要だが、本稿の主張範囲
(組合せ内部の最重要選択である community detection method の妥当性)
の外にあり、future work とする。

\subsection{§8. Conclusions}\label{8-conclusions}

§1 で述べた問題 --- \textbf{AI 出力を関心毎に収束させて認知負荷 / 帯域 /
コストを削減する} --- に対し、§2 の 5 技術組合せが \textbf{task の要件
R1-R5 を充足する設計} であることを §3 で実装、組合せ内部の最重要選択
(community detection method) を §4 で empirical に検証した。

主要な observations (本 domain の本検証範囲での観察):

\begin{enumerate}
\def\labelenumi{\arabic{enumi}.}
\tightlist
\item
  \textbf{Above-limit fraction (Fortunato-Barthélemy ratio) は
  community-detection method に依存しない supplementary K-selection
  signal として機能する}(§4.2)。
\item
  \textbf{本検証範囲 (16 settings + 20 Newsgroups N=100)
  において、Newman backend が CPM backend
  を効果サイズ的に大きく上回った}(H\_struct 87.5\% vs 19\%、20NG ARI
  0.430 vs 0.239 at default K=3、Newman は K=4 強制で 0.557
  まで到達、§4.4 / §4.6)。\textbf{小 sample のため formal
  な統計的有意性検定は実施せず、効果サイズで定性的に報告}する。
\item
  \textbf{CPM の劣化は機序的に説明可能}: \texttt{(n\_c\ choose\ 2)}
  quadratic penalty が under-merging
  を引き起こす。"resolution-limit-free is universally better" は domain
  によらず成立する命題ではない (§5)。
\item
  \textbf{self-routing 指標 (passage を自身の query にした partition
  consistency test) 上、K=10 で 1/K=10\% の理論最小読込量比が両 method
  で成立}(§4.5)。これは production query routing
  の精度を示すものではなく、partition consistency の \textbf{必要条件}
  が満たされたことの報告である。production の paraphrased query routing
  精度、および §6.3 で報告した Mode C の 63\% 実測 token reduction
  との関係は §4.5 / §6.3 内に明示、production 精度確認は future work
  (§7.2 / §7.5)。
\item
  \textbf{Method-selection rule を empirical signals
  から構成可能}(§5.7、詳細は GitHub repository の companion document:
  \url{https://github.com/ChaiCroquis/LayerForge/blob/main/docs/08_empirical_findings.md})。本
  domain では出力がほぼ常に "newman" のため runtime 実装はせず、docs
  のみで保存。
\end{enumerate}

LayerForge は MIT license で公開、Claude Code skill
として運用可能、再現スクリプトと CSV / JSON を同梱。本稿で測定した数値は
publication 時の commit hash で固定される (\texttt{git\ log} 参照)。
